## Orden recomendado de lectura

1. Visión y alcance → entender qué construye el módulo.
2. Arquitectura técnica → decisiones de stack y límites.
3. Modelo de datos → base para backend y API.
4. Estructura de carpetas → convenciones de código.
5. Wireframes + flujo de navegación → UX antes de implementar vistas.
6. Plan de desarrollo → cronograma por etapas.
